코드 보기에 대한 간단한 소개

코드 보기

꽤 많은 양의 글자가 쓰여있지만, 보이는 것처럼 뒤죽박죽이지는 않습니다. 우선, 11행의 <body> 태그를 포함하여 그 위로 보이는 모든 내용은 무시하셔도 됩니다. 태그 내에 써있는 내용들은 HTML 파일에 필요한 정보일 뿐, 일반적으로 변경되어야 하는 부분은 아닙니다.

여러분의 원고는 <body> 태그 뒤부터 시작됩니다.

코드 보기에서는 미리보기 창에 표시된 단어들과 동일한 단어들을 보실 수 있는데요. 즉 "서론"이나 여러 글자 문단을 볼 수 있습니다. 그러나 리더기에게 해당 글자들을 구현하는 방법을 알려주는 HTML 태그들도 함께 볼 수 있습니다.

HTML 기초

HTML에 대한 전체 설명은 이 튜토리얼에서는 포함하지 않았지만, 아래의 기초적인 내용들은 여러분이 원고를 편집하는데 도움이 될 것입니다.[1]

코드 보기에서 파일의 HTML 버전을 보면 <p>나 <h1> 등과 같은 많은 단어들을 볼 수 있습니다. 이것은 당신의 원고에서 글자들이 어떤 역할을 하는지, 전자책 리더기에게 어떻게 표현해야 하는지 알려주는 HTML 언어의 기능이나 정보를 지정하는 태그들입니다.

예를 들면, 아래의 HTML 코드는 Chapter 8이 레벨 1에 해당하는 제목이라고 독자들에게 알려줍니다.

<h1>Chapter 8</h1>

<h1> 태그는 제목처럼 처리해야하는 내용의 시작이며, </h1> 태그는 제목에 포함된 내용의 끝이라는 의미입니다. 여러분이 버튼을 누르면, 시길이 여러분의 원고에 해당 태그를 넣는 방식입니다. 일반적으로 편집자들은 제목의 텍스트를 크게 만들고 화면 중앙에 배치합니다.

아래의 HTML 코드는 리더기에게 This is a paragraph이라는 일반적인 문단을 보여주라고 명령할 것입니다. 리더기에서는 이 단락을 특정 형식으로 보여주는데, 아마도 첫줄은 들여쓰기 하여 나타날 것입니다.

<p>This is a paragraph.</p>

HTML에는 더 많은 태그들이 있습니다. 기억해야 할 중요한 점은 HTML 태그들이 전자책 리더기가 여러분의 원고를 어떤 형식으로 구현하고 표시해야 하는지 알려준다는 것입니다. 이 태그들을 사용하면 내용과 서식을 별도로 유지할 수 있어 책의 모양을 쉽게 변경할 수 있습니다.

HTML 태그들은 속성을 다양하게 수정하여 사용할 수 있기 때문에 더 화려하게 구현될 수 있습니다. 아래와 같이, 마지막 장에 표시된 것처럼 링크 연결에 사용되는 ID를 정의하는 일부 태그를 포함하는 문단과 같은 예가 있습니다

<p><span id="note1">This is a note</span> for linking.</p>

<span> 태그는 단락 내 일부 영역에 다른 형식을 지정하거나 정보를 포함해야 하는 글자들에 사용됩니다. 위와 같은 경우, 여러분이 시길에서 ID추가 버튼을 통해 "This is a note"라는 글자들의 영역에 "note1"이라는 ID 속성이 추가된 <span> 태그를 입력한 것입니다.

HTML 코드의 몇가지 사례에 불과하지만, HTML이 주로 어떤 것에 많이 쓰이는지대한 아이디어를 제공합니다.

중요하게 기억해야할 점은, 코드 보기에서는 여러분이 원고와 책의 모양이 어떻게 표현될 것인가에 대한 서식 코드들을 모두 볼 수 있으므로, 그것들을 찾고 편집할 수 있습니다.

자세한 내용은 소개, 코드 보기 장표와 스타일시트 장표에서 찾아볼 수 있습니다.

코드 보기에서 편집하기

HTML 태그는 아직 변경하지 않은채, 원고의 일부 단어들를 변경해 보고, 미리보기 창에서 변경된 내용을 확인해보세요.

여러분은 꼭 신중하게 편집해야 합니다. 사용하는 모든 태그가 올바르게 사용되었는지, 제대로 열리고 닫혔는지 확인해야 합니다. 그렇지 않으면 미리보기 창에서 분홍색 오류 경고가 표시되고 올바르게 수정될 때까지 ePub파일을 저장할 수 없습니다.

미리보기

코드를 변경하고 실제로 동작하는지 확인하려면 보기=>미리보기 메뉴를 통해 미리보기 창을 열어 확인할 수 있습니다. (보기=>미리보기 메뉴와 단축키 F10을 사용하여 미리보기 창을 닫을 수도 있습니다.) 여러분은 그 창을 더 크게 만들거나 바탕 화면의 다른 부분으로 이동시킬 수도 있습니다.

코드 보기에서 입력하면, 잠시 후 변경 사항이 미리보기에 반영됩니다.

여러분이 작성한 코드를 검사하고 오류를 빠르게 발견하도록 자주 저장하세요.
